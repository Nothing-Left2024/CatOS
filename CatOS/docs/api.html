CatOS API 参考

CatOS 用户程序系统调用完整参考。共 55 个系统调用（NR 0~54）+ 5 个 crt0 内置字符串工具函数，按功能类别分组。每个 API 均包含原型、参数、返回值与可运行示例。

调用约定：用户程序通过 int 0x90 软中断发起系统调用。eax=系统调用号，ebx=第一参数，ecx=第二参数，返回值在 eax。C 代码中只需 extern 声明即可调用，crt0 已封装寄存器传递细节。

三参数函数：file_read/file_write/file_list/file_append/int_to_str 会先调用 set_io_size 传递第三参数（由 crt0 自动处理，C 代码直接调用即可）。

头文件：CatOS SDK 不提供标准头文件，使用前需 extern 声明函数原型。

控制台输出

本组函数用于向控制台输出文本与数值。所有输出均受 set_color 设定的颜色影响，并自动适配文本/图形两种视频模式。

write

原型：int write(char* buf, int len)

NR: 1 | 分类: 控制台输出

描述

将指定长度的字节缓冲区写入控制台输出。按 set_color 设定的当前颜色属性显示，可包含任意字节（含非可见字符）。

参数

  • buf (char*): 要写入的字符串/字节缓冲区指针（段内偏移）
  • len (int): 要写入的字节数

返回值

实际写入控制台的字节数。通常等于 len。

示例

extern int write(char* buf, int len);

static char msg[] = "Hello, CatOS!\n";
write(msg, 14);          /* 输出: Hello, CatOS! */

write("ABC", 3);         /* 输出: ABC */
write("a\tb\n", 4);     /* 输出含制表符与换行 */

备注

buf 是段内偏移，内核自动将其转换为线性地址（USER_PROC_BASE + 偏移）。len 不应超过缓冲区实际长度，否则可能输出段内随机数据。

putc

原型：void putc(int ch)

NR: 2 | 分类: 控制台输出

描述

输出单个字符到控制台，按 set_color 设定的当前颜色显示。

参数

  • ch (int): 要输出的字符 ASCII 码（仅低 8 位有效）

返回值

无（void 函数）。

示例

extern void putc(int ch);

putc('A');       /* 输出: A */
putc('\n');      /* 换行 */
putc(65);        /* 输出: A (ASCII 65) */
putc('0' + 5);   /* 输出: 5 */

备注

图形模式（VGA Mode 13h）下，字符 ASCII <32 或 >127 会显示为 '?'。文本模式下按对应控制字符处理（如 \n 换行、\r 回车、\t 制表符等）。

puts

原型：void puts(char* str)

NR: 3 | 分类: 控制台输出

描述

输出以 \0 结尾的字符串到控制台。不会自动追加换行，需在字符串中显式包含 \n。

参数

  • str (char*): 以 \0 结尾的字符串指针

返回值

无。

示例

extern void puts(char* str);

puts("Hello, CatOS!");          /* 输出后不换行 */
puts("换行需手动添加\n");        /* 输出含换行 */

static char greeting[] = "Hi\n";
puts(greeting);                 /* 使用全局数组 */

备注

str 必须以 \0 结尾，否则会越界读取直到遇到 \0。建议使用全局数组定义字符串以保证可靠性。

set_color

原型：void set_color(int color)

NR: 12 | 分类: 控制台输出

描述

设置后续 putc、puts、write、put_int、put_uint、put_hex、put_bin 输出的默认颜色属性。

参数

  • color (int): 属性字节，高 4 位背景色，低 4 位前景色

返回值

无。

示例

extern void set_color(int color);
extern void puts(char* str);

set_color(0x0F);   /* 黑底亮白 (默认) */
puts("白字\n");

set_color(0x1E);   /* 蓝底黄字 */
puts("黄字\n");

set_color(0x4E);   /* 红底黄字 */
puts("警告\n");

set_color(0x0A);   /* 黑底亮绿 */
puts("成功\n");

备注

常用属性值：0x0F=黑底亮白，0x1E=蓝底黄字，0x4E=红底黄字，0x0A=黑底亮绿，0x0C=黑底亮红，0x0B=黑底亮青。

颜色编码：前景 0~7 暗色，8~15 亮色；背景只能用 0~7。设置后对所有后续输出生效，直到再次调用 set_color。putc_color 不受此设置影响。

putc_color

原型：void putc_color(int ch, int color)

NR: 13 | 分类: 控制台输出

描述

输出一个带指定颜色的字符，不改变 set_color 设定的默认颜色属性。适合在统一文本中插入彩色单字。

参数

  • ch (int): 字符 ASCII 码
  • color (int): 属性字节（高 4 位背景，低 4 位前景）

返回值

无。

示例

extern void putc_color(int ch, int color);
extern void putc(int ch);

/* 彩色输出 "CatOS" */
putc_color('C', 0x0C);   /* 亮红 */
putc_color('a', 0x0A);   /* 亮绿 */
putc_color('t', 0x0B);   /* 亮青 */
putc_color('O', 0x0E);   /* 黄 */
putc_color('S', 0x0D);   /* 亮洋红 */
putc('\n');

备注

调用后默认颜色不变，后续 putc / puts 仍使用之前 set_color 的设置。

put_int

原型：void put_int(int n)

NR: 18 | 分类: 控制台输出

描述

输出有符号十进制整数到控制台。负数前输出 '-'。

参数

  • n (int): 要输出的有符号整数

返回值

无。

示例

extern void put_int(int n);
extern void putc(int ch);

put_int(-42);          /* 输出: -42 */
putc(' ');
put_int(0);            /* 输出: 0 */
putc(' ');
put_int(2147483647);   /* 输出: 2147483647 (INT_MAX) */
putc('\n');

备注

使用当前 set_color 颜色输出。输出后不换行，需手动 putc('\n')。如需将数字写入缓冲区（用于文件等），请使用 int_to_str。

put_uint

原型：void put_uint(unsigned int n)

NR: 19 | 分类: 控制台输出

描述

输出无符号十进制整数。适用于超过 INT_MAX 的大值。

参数

  • n (unsigned int): 要输出的无符号整数

返回值

无。

示例

extern void put_uint(unsigned int n);
extern void putc(int ch);

put_uint(0u);               /* 输出: 0 */
putc(' ');
put_uint(42u);              /* 输出: 42 */
putc(' ');
put_uint(4000000000u);      /* 输出: 4000000000 */
putc('\n');

备注

对于 0x80000000~0xFFFFFFFF 范围的值，若用 put_int 会输出负数，用 put_uint 才能正确显示无符号值。调用时注意加 u 后缀。

put_hex

原型：void put_hex(unsigned int n, int upper)

NR: 20 | 分类: 控制台输出

描述

输出十六进制表示。不含 0x 前缀，需自行添加。

参数

  • n (unsigned int): 要输出的值
  • upper (int): 1=大写 A-F，0=小写 a-f

返回值

无。

示例

extern void put_hex(unsigned int n, int upper);
extern void puts(char* str);

puts("0x"); put_hex(0xDEADBEEFu, 1);   /* 输出: 0xDEADBEEF */
puts(" / 0x"); put_hex(0xdeadbeefu, 0); /* 输出: 0xdeadbeef */
put_hex(0xFF, 1);                       /* 输出: FF */
put_hex(0xFF, 0);                       /* 输出: ff */

备注

0 输出 "0"。不含前导零（例如 0x00FF 输出为 FF）。0x 前缀需调用者自行输出。

put_bin

原型：void put_bin(unsigned int n)

NR: 21 | 分类: 控制台输出

描述

输出二进制表示（不含前导零）。

参数

  • n (unsigned int): 要输出的值

返回值

无。

示例

extern void put_bin(unsigned int n);

put_bin(0);          /* 输出: 0 */
put_bin(1);          /* 输出: 1 */
put_bin(10);         /* 输出: 1010 */
put_bin(255);        /* 输出: 11111111 */
put_bin(0xB4);       /* 输出: 10110100 */

备注

0 输出 "0"，其他值不含前导零。适合底层寄存器/位域调试。

键盘输入

本组函数用于读取键盘输入与修饰键状态。getc 非阻塞且只返回 ASCII；getch 阻塞且返回 16 位键值（含扩展键）；kbhit 检测是否有按键；get_key_flags 获取修饰键状态。

getc

原型：int getc(void)

NR: 4 | 分类: 键盘输入

描述

非阻塞读取键盘。若键盘缓冲区有按键则返回其 ASCII 码，否则立即返回 -1。

参数

无。

返回值

按键的 ASCII 码（0~127），或 -1（无按键）。只返回低 8 位，无法获取扩展键码。

示例

extern int getc(void);
extern void putc(int ch);
extern void exit(int code);

while (1) {
    int ch = getc();
    if (ch >= 0) {
        if (ch == 27) exit(0);
        putc(ch);
    }
}

备注

getc 只返回低 8 位（key & 0xFF），扩展键（方向键、F1~F12 等）会被截断。需要处理扩展键请使用 getch。

getch

原型：int getch(void)

NR: 23 | 分类: 键盘输入

描述

阻塞读取一个按键，直到有按键输入才返回。返回 16 位键值，可区分普通字符与扩展键。

参数

无。

返回值

16 位键值：

  • 普通可见字符（0x20~0x7E）：返回 ASCII 码本身（如 'A'=0x41）
  • 扩展键（Enter/ESC/Tab/Backspace/F1~F12/方向键等）：返回 0x100 + code
  • 修饰键（Shift/Ctrl/Alt 等）：在内部自动跳过，不会返回

示例

extern int getch(void);
extern void exit(int code);

int key = getch();
if (key & 0x100) {
    switch (key) {
    case 0x101: exit(0);          /* ESC */
    case 0x103: break;            /* Enter */
    case 0x104: break;            /* Backspace */
    case 0x125: break;            /* Up */
    case 0x126: break;            /* Down */
    case 0x127: break;            /* Left */
    case 0x128: break;            /* Right */
    }
} else {
    if (key == 'q') exit(0);
}

备注

用 (key & 0x100) 判断是否为扩展键。不要用 & 0xFF 截断，否则方向键等功能键无法识别。常见扩展键值：ESC=0x101，Tab=0x102，Enter=0x103，Backspace=0x104，F1~F12=0x111~0x11C，方向键=0x125~0x128。

kbhit

原型：int kbhit(void)

NR: 24 | 分类: 键盘输入

描述

检测键盘缓冲区是否有按键。不消费按键数据，需配合 getch / getc 读取。

参数

无。

返回值

1 = 有按键待读取，0 = 无按键。

示例

extern int kbhit(void);
extern int getch(void);

while (1) {
    if (kbhit()) {
        int key = getch();
        if (key == 'q') break;
    }
}

备注

典型用法是游戏循环：先 kbhit 检测，有键再 getch 读取，避免阻塞主循环。

get_key_flags

原型：int get_key_flags(void)

NR: 34 | 分类: 键盘输入

描述

获取当前所有修饰键的状态位掩码，可同时检测多个修饰键。

参数

无。

返回值

16 位状态掩码，各位含义：

位掩码含义
00x001左 Shift
10x002右 Shift
20x004左 Ctrl
30x008右 Ctrl
40x010左 Alt
50x020右 Alt
60x040Caps Lock（开关状态）
70x080Num Lock（开关状态）
80x100Scroll Lock（开关状态）

示例

extern int get_key_flags(void);
extern int getch(void);

int key = getch();
int flags = get_key_flags();

if (flags & 0x001) { }
if (flags & 0x002) { }
if (flags & 0x004) { }
if (flags & 0x010) { }
if (flags & 0x040) { }

if ((flags & 0x004) && key == 'c') {
}

备注

Shift/Ctrl/Alt 位反映按键的实时按下状态；Lock 位反映开关的当前状态（按下后切换）。可同时有多个位被置位。

屏幕控制

本组函数用于控制屏幕显示：清屏、光标定位、获取屏幕尺寸与视频模式、滚屏。

clrscr

原型：void clrscr(void)

NR: 6 | 分类: 屏幕控制

描述

清空整个屏幕，并将光标复位到左上角 (0, 0)。

参数

无。

返回值

无。

示例

extern void clrscr(void);
extern void puts(char* str);
extern void gotoxy(int x, int y);

clrscr();
gotoxy(0, 0);
puts("屏幕已清空, 光标回到左上角");

备注

清屏后当前 set_color 颜色设置不变。文本模式下用空格填充；图形模式下清空帧缓冲。

gotoxy

原型：void gotoxy(int x, int y)

NR: 7 | 分类: 屏幕控制

描述

移动光标到指定字符坐标 (x, y)。

参数

  • x (int): 列号（0 ~ get_cols()-1）
  • y (int): 行号（0 ~ get_rows()-1）

返回值

无。

示例

extern void gotoxy(int x, int y);
extern void puts(char* str);

gotoxy(0, 0);          /* 左上角 */
puts("标题行");

gotoxy(0, 12);         /* 屏幕中部 */
puts("中间位置");

gotoxy(30, 24);       /* 底部右侧 */
puts("底部");

备注

坐标超出屏幕范围会被裁剪到边界。常与 set_color 配合实现 UI 布局。

get_xy

原型：int get_xy(void)

NR: 8 | 分类: 屏幕控制

描述

获取当前光标的字符坐标。

参数

无。

返回值

打包后的 32 位整数：(x & 0xFFFF) | (y << 16)。x 在低 16 位，y 在高 16 位。

示例

extern int get_xy(void);
extern void put_int(int n);

int pos = get_xy();
int x = pos & 0xFFFF;
int y = pos >> 16;
put_int(x);
put_int(y);

int saved = get_xy();
gotoxy(saved & 0xFFFF, saved >> 16);

备注

用位运算解析坐标：x = ret & 0xFFFF，y = ret >> 16。

get_cols

原型：int get_cols(void)

NR: 9 | 分类: 屏幕控制

描述

获取屏幕的字符列数（宽度）。

参数

无。

返回值

列数。文本模式 = 80，图形模式（VGA Mode 13h）= 40。

示例

extern int get_cols(void);
extern void put_int(int n);
extern void gotoxy(int x, int y);

int cols = get_cols();
put_int(cols);

static char title[] = "=== CatOS ===";
int len = 13;
gotoxy((cols - len) / 2, 0);
puts(title);

备注

图形模式下由于字体为 8x8 像素，320 像素宽对应 40 列。文本模式标准为 80 列。

get_rows

原型：int get_rows(void)

NR: 10 | 分类: 屏幕控制

描述

获取屏幕的字符行数（高度）。

参数

无。

返回值

行数。通常 = 25。

示例

extern int get_rows(void);
extern void put_int(int n);
extern void gotoxy(int x, int y);

int rows = get_rows();
put_int(rows);

gotoxy(0, rows - 1);
puts("底部行");

备注

标准文本模式 25 行，图形模式（VGA 13h, 200 像素高 / 8 像素字体）也是 25 行。

get_vmode

原型：int get_vmode(void)

NR: 11 | 分类: 屏幕控制

描述

获取当前视频模式。

参数

无。

返回值

0 = 文本模式，1 = 图形模式（VGA Mode 13h, 320x200, 256 色）。

示例

extern int get_vmode(void);
extern void puts(char* str);

if (get_vmode() == 0) {
    puts("当前: 文本模式 (80x25)\n");
} else {
    puts("当前: 图形模式 (320x200, 256色)\n");
}

备注

系统调用输出会自动适配两种模式，无需在程序中特殊处理。图形模式下 putc 对 ASCII <32 或 >127 输出 '?'。

scroll

原型：void scroll(int lines)

NR: 25 | 分类: 屏幕控制

描述

将屏幕内容向上滚动指定行数，底部出现空行。

参数

  • lines (int): 要滚动的行数

返回值

无。

示例

extern void scroll(int lines);
extern void gotoxy(int x, int y);
extern void puts(char* str);

scroll(1);
scroll(5);

gotoxy(0, 24);
puts("新的一行");
scroll(1);

备注

滚动后底部空行使用当前 set_color 的背景色填充。光标位置不随滚动改变。

文件 IO

本组函数提供 FAT12/16/32 文件系统访问。文件名为 8.3 短文件名格式，仅支持根目录下的文件操作。

set_io_size

原型：void set_io_size(int size)

NR: 26 | 分类: 文件 IO

描述

设置后续 file_read / file_write / file_list / file_append / int_to_str 的第三参数。

参数

  • size (int): 字节数（第三参数值）

返回值

无。

示例

extern void set_io_size(int size);
extern int file_read(char* name, char* buf, int bufsize);

set_io_size(4096);
file_read("data.txt", buf, 0);

file_read("data.txt", buf, 4096);

备注

系统调用接口只有两个参数寄存器（ebx/ecx），三参数函数由 crt0 自动先调用 set_io_size 传递第三参数。C 代码中无需手动调用此函数，直接调用 file_read 等即可。

file_read

原型：int file_read(char* name, char* buf, int bufsize)

NR: 27 | 分类: 文件 IO

描述

读取文件内容到缓冲区。从文件开头读取，最多 bufsize 字节。

参数

  • name (char*): 8.3 短文件名（如 "hello.ce"、"data.txt"）
  • buf (char*): 接收数据的缓冲区
  • bufsize (int): 最大读取字节数

返回值

实际读取的字节数；文件不存在或失败返回 -1。

示例

extern int file_read(char* name, char* buf, int bufsize);
extern void puts(char* str);
extern void put_int(int n);

static char buf[4096];
int n = file_read("test.txt", buf, sizeof(buf));
if (n < 0) {
    puts("文件不存在\n");
} else {
    buf[n] = '\0';
    puts("读取 "); put_int(n); puts(" 字节:\n");
    puts(buf);
}

备注

文件名不区分大小写，内核内部转大写匹配。bufsize 决定最大读取量，缓冲区必须在用户段内（受 128KB 段大小限制）。

file_write

原型：int file_write(char* name, char* buf, int size)

NR: 28 | 分类: 文件 IO

描述

写入文件。文件不存在时创建；已存在时覆盖原有内容。

参数

  • name (char*): 8.3 文件名
  • buf (char*): 要写入的数据缓冲区
  • size (int): 要写入的字节数

返回值

实际写入的字节数；失败返回 -1。

示例

extern int file_write(char* name, char* buf, int size);
extern void put_int(int n);

    put_int(written);
}

备注

会覆盖原有内容。如需在文件末尾追加请使用 file_append。文件大小受 FAT 文件系统和磁盘空间限制。

进程管理

spawn

原型：int spawn(char* name)

NR: 39 | 分类: 进程管理

描述

启动另一个程序（非阻塞）。新程序在独立的进程槽位中运行，调用者立即返回继续执行。

参数

  • name (char*): 要启动的程序文件名（如 "SHELL.BIN"）

返回值

成功返回新进程的槽位号（>=0）；失败返回错误码（<0），常见原因包括文件不存在、进程槽位已满。

} else {
    puts("启动失败\n");
}

备注

非阻塞调用：新进程创建后立即返回，不等待其结束。CatOS 进程槽位有限，请合理使用。被启动程序需位于文件系统根目录。

win_get_event

原型：int win_get_event(int win_id, int* type, int* x, int* y)

NR: 46 | 分类: 窗口 API

描述

阻塞获取指定窗口的事件。若无事件，调用线程会被挂起直到有事件到来。

参数

  • win_id (int): 窗口 ID
  • type (int*): 输出事件类型
  • x (int*): 输出事件的 X 坐标（鼠标事件为客户区相对坐标，键盘事件为按键值）
  • y (int*): 输出事件的 Y 坐标

返回值

成功返回事件类型（见下表）；无事件时阻塞，不会返回 0。

事件类型

常量值说明x 字段y 字段
EV_NONE0无事件--
EV_LDOWN1左键按下鼠标 x鼠标 y
EV_LUP2左键释放鼠标 x鼠标 y
EV_MOUSE3鼠标移动鼠标 x鼠标 y
EV_KEY4键盘按键按键值0
EV_CLOSE5关闭请求--

示例

extern int win_get_event(int win_id, int* type, int* x, int* y);
extern void win_draw_rect(int win_id, int x, int y, int w, int h, int color);

int type, x, y;
while (1) {
    win_get_event(win, &type, &x, &y);
    if (type == 1) {            /* EV_LDOWN */
        win_draw_rect(win, x, y, 5, 5, 0x0C);  /* 画红点 */
    } else if (type == 5) {     /* EV_CLOSE */
        break;
    }
}

备注

阻塞调用：无事件时当前进程会被挂起，不占用 CPU。鼠标坐标为相对于窗口客户区的坐标。EV_CLOSE 通常由用户点击窗口关闭按钮触发，程序应在收到后调用 win_close 或 exit。

  • x (int): 像素 X 坐标
  • y (int): 像素 Y 坐标
  • color (int): 像素颜色

返回值

无（void 函数）。

示例

extern void win_set_pixel(int win_id, int x, int y, int color);

/* 在 (50,50) 画一个白点 */
win_set_pixel(win, 50, 50, 0x0F);

/* 画一条手工水平线 */
for (int i = 0; i < 100; i++) {
    win_set_pixel(win, i, 20, 0x0C);
}

备注

坐标超出客户区边界时该像素被忽略（不报错）。频繁单像素绘制性能较低，批量绘制建议使用 win_draw_rect 或 win_draw_line。

win_peek_event

原型：int win_peek_event(int win_id, int* type, int* x, int* y)

NR: 51 | 分类: 窗口 API

描述

非阻塞获取指定窗口的事件。无论是否有事件都立即返回。

参数

  • win_id (int): 窗口 ID
  • type (int*): 输出事件类型
  • x (int*): 输出事件的 X 坐标
  • y (int*): 输出事件的 Y 坐标

返回值

有事件返回事件类型（>0）；无事件返回 EV_NONE（0）。事件类型同 win_get_event。

事件类型

常量值说明x 字段y 字段
EV_NONE0无事件--
EV_LDOWN1左键按下鼠标 x鼠标 y
EV_LUP2左键释放鼠标 x鼠标 y
EV_MOUSE3鼠标移动鼠标 x鼠标 y
EV_KEY4键盘按键按键值0
EV_CLOSE5关闭请求--

示例

extern int win_peek_event(int win_id, int* type, int* x, int* y);
extern void msleep(int ms);

int type, x, y;
while (1) {
    int ev = win_peek_event(win, &type, &x, &y);
    if (ev == 5) {            /* EV_CLOSE */
        break;
    }
    /* 做其他工作（如动画刷新） */
    msleep(33);               /* ~30 FPS */
}

备注

非阻塞：无事件时立即返回 0，适合游戏循环/动画场景，与定时刷新配合使用。事件类型含义与 win_get_event 相同。

win_get_size

原型：int win_get_size(int win_id, int* w, int* h)

NR: 52 | 分类: 窗口 API

描述

获取指定窗口的客户区尺寸（像素）。

参数

  • win_id (int): 窗口 ID
  • w (int*): 输出客户区宽度
  • h (int*): 输出客户区高度

返回值

0 = 成功；<0 = 失败（无效 win_id）。

extern int win_get_size(int win_id, int* w, int* h);

int w, h;

win_set_closable

原型：void win_set_closable(int win_id, int closable)

NR: 53 | 分类: 窗口 API

描述

设置用户窗口是否允许通过点击标题栏叉号（×）关闭。

参数

  • win_id (int): 窗口 ID（win_create 返回值）
  • closable (int): 0=禁止叉号关闭，1=允许叉号关闭（默认值）

返回值

无（void 函数）。

示例

extern int win_create(int x, int y, int w, int h, char* title);
extern void win_set_closable(int win_id, int closable);
extern int win_get_event(int win_id, int* type, int* x, int* y);

int win = win_create(100, 50, 200, 140, "Form");

/* 表单未提交时禁止用户通过叉号关闭 */
win_set_closable(win, 0);

/* ... 完成表单提交后允许关闭 ... */
win_set_closable(win, 1);

备注

默认情况下，所有窗口都允许通过叉号关闭（点击叉号会推送 EV_CLOSE 事件给应用）。设置 closable=0 后，点击叉号不会产生任何效果，窗口无法通过叉号关闭。应用仍可通过 win_close() 主动关闭窗口。典型用途：强制用户完成操作后才能关闭窗口（如表单未提交时禁止关闭）。

win_get_closable

原型：int win_get_closable(int win_id)

NR: 54 | 分类: 窗口 API

描述

查询用户窗口是否允许通过点击叉号关闭。

参数

  • win_id (int): 窗口 ID

返回值

1 = 允许叉号关闭，0 = 禁止叉号关闭（或窗口不存在）。

示例

extern int win_get_closable(int win_id);
extern void win_set_closable(int win_id, int closable);
extern void put_int(int n);

/* 查询并切换窗口的叉号关闭状态 */
int cur = win_get_closable(win);
put_int(cur);
win_set_closable(win, !cur);

备注

查询的是 win_set_closable 设定的状态，默认值为 1（允许关闭）。传入无效 win_id 返回 0，调用方无法区分“禁止关闭”与“窗口不存在”，应先通过其他方式确认窗口有效性。

窗口关闭与进程退出行为

不同类型的窗口在叉号关闭行为与进程退出时遵循以下规则：

  • 应用主窗口（用户控制台窗口）：点击叉号会直接终止进程，并自动关闭该进程创建的所有子窗口。
  • 子窗口（win_create 创建的）：点击叉号默认推送 EV_CLOSE 事件给应用，应用可在事件循环中处理。如需禁用此行为，可调用 win_set_closable(win_id, 0)，设置后点击叉号不会产生任何事件。
  • 应用主动调用 exit(code) 退出：会自动关闭该进程创建的所有子窗口并终止进程，退出码记录在进程的 exit_code 中。

类型转换

本组函数用于整数与字符串之间的相互转换。比 put_int 等更灵活，可写入缓冲区供文件 IO 或字符串拼接使用。

int_to_str

原型：int int_to_str(int value, char* buf, int radix)

NR: 36 | 分类: 类型转换

描述

将整数转换为指定进制的字符串表示。

参数

  • value (int): 要转换的整数
  • buf (char*): 输出缓冲区，至少 33 字节（二进制 32 位 + \0）
  • radix (int): 进制（2~36），0 默认按 10 进制

返回值

写入的字符数（不含 \0）。

示例

extern int int_to_str(int value, char* buf, int radix);
extern void puts(char* str);

char buf[33];

int_to_str(42, buf, 10);
int_to_str(-123, buf, 10);
int_to_str(255, buf, 16);
int_to_str(10, buf, 2);
int_to_str(511, buf, 8);
puts(buf);

备注

radix=10 时支持负数（前缀 -）；其他进制按无符号处理。0 输出 "0"。十六进制使用小写 a-f。比 put_int 更灵活——结果写入缓冲区，可用于写文件、字符串拼接。

atoi

原型：int atoi(char* str)

NR: 37 | 分类: 类型转换

描述

将字符串转换为整数（十进制解析）。

参数

  • str (char*): 要解析的字符串

返回值

解析得到的整数。无有效数字时返回 0。

示例

extern int atoi(char* str);
extern void put_int(int n);

int a = atoi("42");
int b = atoi("  -7");
int c = atoi("+100");
int d = atoi("123abc");
int e = atoi("0x1F");
int f = atoi("abc");
put_int(a + b);

备注

跳过前导空白字符，处理可选的 +/- 符号，解析到第一个非数字字符停止。无有效数字返回 0。有溢出保护，溢出时饱和到 INT_MAX / INT_MIN。仅支持十进制。
